## 1、基本概念

>基数定理
>
>设度数为 0、1、2 的节点分别有 $n_0, n_1, n_2$ 个：
>
>$$n_0 = n_2 + 1$$
>



-   深度（depth）：距离根节点的路径长度就是深度

-   高（height）：节点的高度 = 后代的最大深度 - 自己的深度

>   所以叶子节点的高为`0`



-   真二叉树：没有度为1的二叉树

-   完全二叉树：除了最后一层外，其他层节点都是满的，并且最后一层的节点尽可能靠左排列



## 2、遍历中的"藤"

**藤**：从某节点出发，沿 **左孩子** 一路向下的通路（左侧通路/长子通路）。

迭代式遍历的核心思路都基于藤的分解：

### 先序遍历（迭代）

-   藤：从根一路向左，**自上而下**访问藤上节点，右孩子压栈待处理
-   关键：只有右孩子才入栈，`#push = n+1`

### 中序遍历（迭代）

-   藤：将藤上节点**自顶而下压栈**，到底后弹出访问，再转向右子树
-   `EOV(x)`：End Of Vine，沿左孩子到底，即中序首节点/右子树中序后继
-   `LLA(x)`：Lowest Left Ancestor，无右子树时向上找**最低左祖先**作为后继

### 后序遍历（迭代）

-   藤的构造：**尽可能向左，实不得已才向右**，目标是找到最靠左的叶节点

-   `gotoLeftmostLeaf(S)`：右孩子优先入栈，再走左孩子

    

**空间复杂度**都是$O(h)$

## 3、二叉树重建

| 序列组合    | 能否唯一重建                         |
| ----------- | ------------------------------------ |
| 先序 + 中序 | ✅ 分治，用中序定根位置               |
| 后序 + 中序 | ✅ 同上                               |
| 先序 + 后序 | ❌ 一般不唯一；除非是在**真二叉树**下 |


**增强序列**：遍历时将 `nullptr` 也输出为 `^`，则先序/后序增强序列可唯一重建（中序不行）。

>   已知一棵二叉树的先序和后序遍历序列如下：
>
>   -   **先序遍历 (Pre-order):** `[1, 2, 3, 4]`
>   -   **后序遍历 (Post-order):** `[4, 3, 2, 1]`
>
>   **请问：**
>
>   1.  该树有多少种可能的物理结构？
>   2.  这棵树的**层序遍历**序列是什么？



>**先序：** `[1, 2, 3, 4, 5]`
>
>**后序：** `[3, 2, 5, 4, 1]`
>
>可以确定层次但是不能确定`3` `5`的位置



### 卡特兰数

这三者是完全等价的：

-   **$n$ 个节点的不同二叉树形态数**
-   **$n$ 个元素的合法出栈序列数（栈混洗）**
-   **$n$ 对括号的合法匹配方式**

>   所以如果给定了一种序列的遍历，另一个序列能确定二叉树形状。则另一个序列的数量为卡特兰数，且序列一定对应一个栈混洗。



## 4、Huffman 编码树

### 核心概念

-   **前缀无关编码（PFC）**：字符对应叶节点，根到叶的路径即编码，天然无歧义
-   **带权编码长度（WCL）**= 所有内部节点的权重之和 = $\sum freq(x) \times depth(x)$
-   **最优编码树**（Optimal Coding Tree）：**OCT** 通常也被称为 **哈夫曼树**（Huffman Tree），是一种带权路径长度（WPL）最短的二叉树。它在数据压缩、信息论和算法设计中有着极其重要的地位。
-   **最优带权编码树（OWCT）**：WCL 最小的编码树

### Huffman 算法（贪心，1952）

每次从森林中取出**权重最小的两棵树**合并，直到只剩一棵。

**正确性关键引理**：频数最低的两个字符，在某棵 OWCT 中**必为兄弟**。

### 复杂度

| 实现                | 复杂度                  |
| ------------------- | ----------------------- |
| 列表/向量           | $O(n^2)$                |
| 完全二叉堆 / 左式堆 | $O(n \log n)$           |
| 预排序 + 栈 + 队列  | $O(n \log n)$，常数更优 |
