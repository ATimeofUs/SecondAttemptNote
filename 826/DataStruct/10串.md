### 1.2 kmp

**定义：**

-   p 是 pattern 字符串, t 是 text 字符串

-   next[j]：p[0, j - 1]这个段最长的、公共的、前后缀的长度

**创建next代码**

-   t,j 在进入if的正确条件的时候，一定保持着p[0 ~ t-1] == p[xxx ~ j-1]的前后缀相等

```c
int t = -1;
next[0] = -1;

while (j < m - 1){
    if (t < 0 || p[t] == p[j]){
        t ++, j ++;
        next[j] = t;
    }else{
        t = next[t];
    }
}
```

**匹配代码：**

```c
while (j < m && (i - j) <= (n - m)){
    if (j > 0 || t[i] == t[j]){
        i ++, j ++;
    }else{
        j = next[j];
    }
}

return i - j;

```

改良版本`next`的把字符相同且存在 `next` 指向关系的节点用并查集 `merge` 起来，最后通过 `next[fa[i]]` 实现路径压缩，运行结果完全一致。

`next[i]`和`i`合并规则如下
$$
next[i] \ge 0 \space and \space p(i) = p(next[i])
$$


## 2. BM算法

原名：[Boyer + Moore, 1977] A Fast String Searching Algorithm

1.   预处理：根据模式串P，预先构造bc[]表（以及gs[]表） 与next[]类似，为日后的局部适配提供快捷处置的建议
2.   迭代：自右向左依次比对字符，找到极大的匹配后缀 若完全匹配，则返回位置；否则 
     -   查表确定P右移的适当位置
     -   重新自右向左比对

### 2.1 BC策略：坏字符

原名：Bad-Character Shift

#### 2.1.1 算法原理：

>   1.   i 尝试移动滑动窗口，使得下次匹配刚好有一个位置可以匹配这个坏字符
>
>   2.   j 从尾巴开始
>
>   3.   匹配仍然是从头开始匹配



#### 2.1.2 复杂度

最好时间复杂度 $O(n / m)$

-   只要`P`没有`T[m - 1]`，每次移动都是**m格**

最差时间复杂度 $O(n \cdot m)$

-   `P[0]`和 `T[0]`不相同，其他都相同，每次只能移动**1格**

### 2.2 BM算法：GS策略



![image-20260608151701176](/home/ping/Second_Attempt/826/DataStruct/assets/image-20260608151701176.png)

**$O$ (Old/字符)**：模式串中当前卡住的位置 $P[j]$。

**$T$ (Target/坏字符)**：文本串中对应的字符 $T[i+j]$。因为比对中断，所以 $T \neq O$。

**$U$ (好后缀)**：模式串末尾已经匹配成功的那段子串 $P(j, m)$



🟢 条件一：保持经验（$U$ 与 $V(k)$ 匹配）

-   既然右边的子串 $U$ 已经在文本串里匹配成功了，那我们把模式串往右挪动后，**新对齐的那部分子串（记为 $V(k)$），内容必须长得和 $U$ 一模一样**。否则落下来也肯定匹配不上。

🔴 条件二：教训汲取（$P[k] \neq O$）

-   因为 $P[j] = O$，而文本串那里是 $T$，因为 $O \neq T$ 所以失败。 这说明**文本串那个位置需要的是 $T$，绝对不是 $O$**！



匹配时：

完全匹配：在这样的k中，取最大（最靠后）者 //移动距离最小 

部分匹配：若没有这样的k，则在所有前缀P[0,t)中，取与U匹配的最长者 //有可能t=0 

无论如何，位移量仅取决于j和P本身——亦可预先计算，并制表待查



### 2.3 GS

#### 2.3.1 SS数组

`bbaaba`的SS数组

| **索引 i** | **字符 P[i]** | **以 i 结尾的子串** | **SS[i] 值**       |
| ---------- | ------------- | ------------------- | ------------------ |
| **0**      | `b`           | `"b"`               | **0**              |
| **1**      | `b`           | `"bb"`              | **0**              |
| **2**      | `a`           | `"bba"`             | **2** (`"ba"`)     |
| **3**      | `a`           | `"bbaa"`            | **1** (`"a"`)      |
| **4**      | `b`           | `"bbaab"`           | **0**              |
| **5**      | `a`           | `"bbaaba"`          | **6** (`"bbaaba"`) |

#### 2.3.2 GS数组

定义：当我们在模式串的 $j$ 位置比对失败时（此时好后缀为 $P[j+1 \dots m-1]$），模式串可以安全向右滑动的最大距离

1.   ss[j] = j + 1，$i < m - j - 1$，j是gs[i]候选

2.   ss[j] < j + 1, $i = m - j - 1$，i是gs[i]的候选

#### 2.3.3 复杂度

最好时间复杂度 $O(n / m)$

最差时间复杂度 $O(n \cdot m)$

单次匹配的概率 = 1 / 字符集数量
