

# 弱智基础

## 1. 边的分类：有向图 vs 无向图

竞赛中我们通常只关心“树边”和“非树边”，但邓老师课件中对 DFS 边类型的划分非常细致，这是考研常考点：

-   有向图的四种边： `TREE`（树边）、`BACKWARD`（后向边）、`FORWARD`（前向边）、`CROSS`（跨边）。
-   无向图 DFS 只存在 `TREE` 和 `BACKWARD`。竞赛选手习惯用有向边存无向图，容易混淆。

-   dTime (发现时间，Discovery Time)
-   fTime (完成时间，Finish Time) 

-   **判别准则：**
    -   **后向边：** 指向活跃期还没结束的祖先（`status == DISCOVERED`），换句话就是说还在dfs堆栈当前的节点
    -   **前向边 vs 跨边：** 指向的节点是 `VISITED` 节点。区别在于 `dTime`：
        -   `dTime(v) < dTime(u)` 是前向边（指向子孙）。
        -   `dTime(v) > dTime(u)` 是跨边（指向已完成的无关分支）。

## 2. 顶点状态位

顶点有三种状态：

-   **UNDISCOVERED**：还没见过。
-   **DISCOVERED**：已发现，在栈/队列中，活跃期开始（记录 `dTime`）。
-   **VISITED**：已访问完毕，所有邻接点处理完，活跃期结束（记录 `fTime`）。

## 3. BFS 中的“跨边”（CROSS Edge）

在无向图中，BFS 也会产生 **CROSS 边**。

-   **规律：** BFS 树中由跨边 $(v, u)$ 连接的顶点，其到源点的距离 `dist(s, v)` 与 `dist(s, u)` 的**差值绝对值至多为 1**。
-   **应用：** 这一点常用于证明 BFS 求解最短路的正确性，或者判断图是否为**二部图**（奇数长度的环一定包含 $dist$ 相同的 CROSS 边）。
