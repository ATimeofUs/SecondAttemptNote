# 向量

## 1. 二分查找

**查找长度**是指：在查找算法执行过程中，**所执行的“元素大小比较操作”的总次数**。

>   **文中的原话：** “查找算法的整体效率也更主要地取决于其中所执行的元素大小比较操作的次数，即所谓查找长度。”

作者在这里做了一个非常实用的性能分析。在一次二分查找的迭代中，其实有两种操作：

1.  **秩的运算**：比如 `mid = (lo + hi) >> 1`，这是整数算术，CPU跑起来极快。
2.  **元素比较**：比如 `target < S[mid]`。如果向量里的元素是复杂的对象（比如长字符串、大结构体），这种比较会非常耗时。

一般来说看2更加好

#### 二分查找a版本

``````
while ( lo < hi ) { //殏步迭代可能要做两次比较刞断，有三个分支
	int mi = ( lo + hi ) >> 1; //以中点为轴点
    
    if ( e < A[mi] ){
		 hi = mi;
	}
    else if ( A[mi] < e ){
		lo = mi + 1;
	} 
    else{
		return mi;
	} 
} 
``````

时间复杂度 $O(1.5 \cdot logn)$ ，因为平均需要比较1.5次

#### 二分查找版本c

```c
int binSearch(int * S, int e, int l, int h ) {
    while ( l < h ) { //不变性：[0, lo) <= e < [hi, n)
    	int mid = (lo + h) / 2;
    	if ( e < S[mid] ) hi = mid; //[lo, mi)
    	else l = mid + 1; //(mi, hi)
    } //出口时，[lo = hi] = M
    
    return lo - 1; //至此，[lo]为大于e的最小者，故[lo-1] = m即为不大于e的最大者
}
```

## 2. 插值查找

和二分不一样, 每次查找的`mid`为 
$$
\frac{mid - l}{r - l} = \frac{target - a[l]}{a[r] - a[l]}
$$
