## 1. 单周期CPU

每条指令的执行过程
- 第一步：取指令（IF）
- 第二步：指令译码（ID）
- 第三步：执行指令（EXE）
- 第四步：访问存储器（MEM）
- 第五步：写回寄存器（WB）



计算机一条指令的执行时间被称为**指令周期**，一个CPU时钟时间被称为**CPU周期**(在某些计算机中，还可再把一个CPU周期区分为几个更小的步骤，称其为节拍)。执行每条指令平均使用的CPU周期个数被称为**CPI**

-   定义：全部指令都选用一个CPU周期完成的系统被称为单周期CPU

优点

- 每条指令占用一个时钟周期
- 逻辑设计简单，时序设计也简单

缺点

- 各组成部件的利用率不高
- 维持有效信号
- 时钟周期应满足执行时间最长指令的要求 —— Load指令

## 2. 多周期CPU

将指令执行过程分解成多个步骤，每条指令占用它需要的步骤数 

每个步骤占用**一个时钟周期** 

-   尽量平衡各步骤间的延迟 

-   尽量限制每个步骤使用单一的主要部件 

-   控制器仅需提供当前步骤所需要的控制信号

基本前提

- 保存好下一步骤要用到的值，需要引入“新”的内部寄存器（**锁存器，寄存器**）
- 引入**有限自动机**标记当前步骤



### 2.1. 两种不同类型的多周期控制器

硬布线控制器和微程序控制器

#### 2.1.1 微程序控制器核心原理

-   采用控制存储器存储每条指令的每个执行步骤所需要的全部控制信号，用微地址进行访问，读出控制信号并输出

#### 2.1.2 下地址逻辑

采用下地址逻辑实现执行步骤之间的衔接：
  - 根据指令操作码映射出该指令的首条微指令的地址
  - **每条微指令给出其下一步骤的微地址**

## 3. CPU 控制器

### 3.1 基本组成

- 协调并控制计算机各部件执行程序的指令序列
- 基本功能包括**取指令、分析指令、执行指令**
    - 取指令：自动形成指令地址；自动发出取指令的命令
    - 分析指令：操作码译码（分析本条指令要完成什么操作），产生操作数的有效地址
    - 执行指令：根据分析指令得到的“操作命令”和“操作数地址”，形成操作信号控制序列，控制运算器、存储器以及I/O设备完成相应的操作
- 中断处理：管理总线及输入输出；处理异常情况（如掉电）和特殊请求（如打印机请求打印一行字符

| 程序计数器       | PC (Program Counter)          | 指出下一条指令在主存中的地址；具备自增功能或跳转能力      | 与主存地址位数相同           |
  | ---------------- | ----------------------------- | --------------------------------------------------------- | ---------------------------- |
  | 部件名称         | 缩写                          | 功能描述                                                  | 位数依据                     |
  | 指令寄存器       | IR (Instruction Register)     | 保存当前正在执行的指令                                    | 与指令字长相同               |
  | 指令译码器       | ID (Instruction Decoder)      | 对 IR 中的操作码字段进行译码，确定操作功能                | 与操作码位数相关             |
  | 存储器地址寄存器 | MAR (Memory Address Register) | 存放要访问的主存单元地址                                  | 与主存地址线位数相同         |
  | 存储器数据寄存器 | MDR (Memory Data Register)    | 存放从主存读出或写入主存的数据                            | 与存储字长相同               |
  | 时序系统         | —                             | 产生各种时序控制信号，通常由统一时钟分频生成              | 依赖系统时钟频率与控制逻辑   |
  | 微操作信号发生器 | —                             | 根据 IR、PSW 和时序信号生成控制信号，驱动各部件执行微操作 | 有组合逻辑型与存储逻辑型两类 |

### 3.2 CPU 常混淆点

#### 3.2.1 各寄存器的位数

【和地址有关的就取决于机器字长，和数据大小有关的就取决于容量】

- 通用寄存器：机器字长
- PC：看存储器能装下多少指令
- IR：指令字长
- MAR：存储器容量
- MDR：存储字长

#### 3.2.2 周期

-   **时钟周期（节拍 / T 周期）**：计算机工作的最小时间周期，是 CPU 操作的基本单位；一个时钟周期内控制信号不发生改变
-   **机器周期（CPU 周期）**：指 CPU 完成一个基本操作（如取指、译码、执行、访存、写回等）所需的时间

- **指令周期**：CPU 从主存中取出并执行一条指令的时间，一个指令周期由多个机器周期组成

    存储周期与上面独立存在

- **存取周期**：连续启动两次独立的读/写操作所需要的最短时间



## 4、指令流水线

### 4.1、流水线的基本概念

- **取指（IF）**：从指令存储器中取出下一条指令，并送入流水线。
- **译码（ID）**：对指令进行译码，识别操作类型，同时从寄存器堆中取出源操作数，立即数在这里扩展。
- **执行（EX）**：由 ALU 完成具体运算，或者进行地址计算（如访存指令的目标地址）。
- **访存（MEM）**：如果指令需要访问数据存储器，就在这一阶段进行读或写操作；算术指令在此阶段通常是空操作。
- **写回（WB）**：将结果写回寄存器堆，更新目标寄存器的值。



#### 4.1.1 流水线的性能指标

##### 1）流水线的吞吐率（TP）

- **吞吐率**：单位时间内流水线所完成的任务数量，或是输出结果的数量

- 设任务数 m，处理完成 n 个任务所用的时间为 $T_k$

- $TP=\frac{n}{T_k}=\frac{n}{(k+n-1)Δt}$ 

- 最大吞吐率：$TP_{max}=\frac{1}{Δt}$

    


- **注意**：
    - m 段流水线的 CPU 吞吐能力 = m 个并行部件的 CPU 吞吐能力
    - m 段流水线在第 m 个时钟周期后，每个时钟周期都可完成一条指令
    - m 个并行部件在 m 个时钟周期后能完成全部的 m 条指令，等价于平均每个时钟周期完成一条指令

##### 2）流水线的加速比（S）

- **加速比**：完成同样一批任务，**不使用流水线所用的时间**与**使用流水线所用的时间**之比
- 设顺序执行的时间为 $T_0$，使用流水线的执行时间 $T_k$
- $S=\frac{T_0}{T_k}=\frac{kn}{k+n-1}$
- 最大加速比： $S_{max}=k$


##### 3）流水线的效率（E）

- **效率**：流水线的设备利用率，在时空图上，表示为**完成 n 个任务占用的时空区有效面积**与 **n 个任务所用的时间与 k 个流水段所围成的时空区总面积**之比
- $E=\frac{T_0}{kT_k}$

### 4.2 流水线的基本实现

#### 4.2.1 流水线的数据通路

- **数据通路**：数据在功能部件之间传送的路径
- 包括数据通路上流经的部件（如 PC，ALU，通用寄存器，状态寄存器，异常和中断处理逻辑）
- **数据通路由控制部件控制**，控制部件根据每条指令功能的不同生成对数据通路的控制信号
- **数据通路不包含生成控制信号的控制部件** 

| 阶段缩写 | 阶段名称           |
| -------- | ------------------ |
| IF       | Instruction Fetch  |
| ID       | Instruction Decode |
| EX       | Execute            |
| MEM      | Memory Access      |
| WB       | Write Back         |

#### 4.2.2 流水线的控制信号

- **流水寄存器保存的信息**
    - 后面流水段要用到的**所有数据信息**
        - 包括 PC+4,、指令、立即数、目的寄存器、ALU 运算结果、标志信息等
        - 它们是前面阶段在数据通路中执行的结果
    - 后面传递过来的后面各流水段要用到的**所有控制信号**

#### 4.2.3 流水线的执行过程

- 假设一条指令的执行有 5 条流水线
    - **取指（IF）**：从指令寄存器或 Cache 中取指令
    - **译码/读寄存器（ID）**：操作控制器对指令进行译码，同时从寄存器堆中读取操作数
    - **执行/计算地址（EX）**：执行运算操作或计算地址
    - **访存（MEM）**：对存储器进行读写操作
    - **写回（WB）**：将指令执行结果写回寄存器

### 4.3 流水线的冒险与处理

- **流水线冒险**：在指令流水线中，可能会遇到一些情况使得后续指令无法正确执行而引起流水线阻塞的现象。根据导致冒险的原因不同分为：结构冒险、数据冒险和控制冒险。解决流水线冒险的通用方法就是放弃流水线——暂停流水线，可以分为以下方式：

    - 软件方式：在编译时期，插入空操作nop。硬件控制简单但是浪费了很多空间和时间。
    - 硬件方式：插入气泡。通过硬件实现，这种方式控制比较复杂，需要修改数据通路。

    其实不怎么重要，只要计算时钟周期即可（

#### 4.3.1 结构冒险

- 定义：由于多条指令在同一时刻争用同一资源而形成的冲突，也称**资源冲突**。

- 解决方法：

    【1】前一指令访存时，使后一条指令（以及其后续指令）**暂停一个时钟周期**

    【2】设置多个独立的部件：

    - 寄存器访存冲突：将寄存器的读口和写口独立开来
    - 访存冲突：单独设置数据存储器和指令存储器，使取数和取指令操作各自在不同的存储器中进行
    - **双沿访问**：寄存器访问分为两步，第一步在前半个时钟周期准备写出，然后在下降沿写入；第二步在后半时钟周期读出。可以节约一个时钟周期，这是因为寄存器访问很快。（寄存器默认采用的方法）

#### 4.3.2 数据冒险

- **定义**：后面指令用到前面指令的结果时，前面指令的结果还没有产生，也称**数据相关**
- 乱序执行的流水线中存在三种数据相关：
    - **写后读 RAW（只有RISC会发生）**：当前指令将数据写入寄存器后，下一条指令才能从该寄存器读取数据
    - 读后写 WAR：当前指令读出数据后，下一条指令才能写入寄存器 
    - 写后写 WAW：当前指令写入寄存器后，下一条指令才能写入寄存器 

#### 4.3.3 控制冒险

- **定义**：当流水线遇到转移指令和其他改变 PC 值的指令而造成断流时，会引起**控制相关**



### 4.4 流水线的分类

| **分类**                                          | **描述**                                                     | **举例**                                                     |
| ------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **部件功能级流水线**                              | 将复杂的算术逻辑运算分成多个子过程，以流水线方式执行。       | 例如浮点加法可分为：求阶差、对阶、尾数相加、结果规格化等四个阶段。 |
| <span style="color:red">**处理机级流水线**</span> | 将指令的执行过程分成多个子过程，如：取指（IF）、译码（ID）、执行（EX）、访存（MEM）、写回（WB）五个阶段。 | MIPS、ARM、x86 等处理器的指令流水线                          |
| **处理机间流水线**                                | 一种宏流水结构，每个处理机完成专门任务，结果存放在与下一个处理机共享的存储器中。 | 图像处理流水线、工业自动化中的多处理机协作系统               |



| **分类**                                        | **描述**                                             | **举例**                                      |
| ----------------------------------------------- | ---------------------------------------------------- | --------------------------------------------- |
| **单功能流水线**                                | 只能实现一种固定专门功能的流水线。                   | 专用加法流水线、视频编码器中的固定功能模块    |
| <span style="color:red">**多功能流水线**</span> | 通过不同连接方式，可同时或分别实现多种功能的流水线。 | 可配置处理器流水线、FPGA 中的多功能流水线设计 |



| **分类**       | **描述**                                                     | **举例**                               |
| -------------- | ------------------------------------------------------------ | -------------------------------------- |
| **静态流水线** | 同一时间内，所有流水线段只能按同一种功能连接方式工作。       | MIPS 指令流水线、传统 RISC 架构处理器  |
| **动态流水线** | 同一时间内，不同流水线段可执行不同运算，提高效率但控制复杂。 | 超标量处理器、动态调度的乱序执行流水线 |



| **分类**         | **描述**                                               | **举例**                                     |
| ---------------- | ------------------------------------------------------ | -------------------------------------------- |
| **线性流水线**   | 每个功能段只经过一次，无反馈回路。                     | 指令执行流水线、图像处理中的线性数据流       |
| **非线性流水线** | 存在反馈回路，某些功能段可多次参与运算，适合迭代计算。 | 数值迭代算法流水线、神经网络训练中的反馈结构 |

### 4.5 五段式流水线

| 指令类型     | IF（取指）        | ID（译码）                                 | EX（执行）                    | M（访存）                      | WB（写回）           |
| ------------ | ----------------- | ------------------------------------------ | ----------------------------- | ------------------------------ | -------------------- |
| **STORE**    | 从指令Cache取指令 | A ← 基址寄存器，Imm ← 偏移量，B ← 要存的值 | 有效地址 ← A + Imm，Store ← B | 将 Store 写入数据 Cache        | 空段                 |
