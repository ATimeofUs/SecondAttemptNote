# 一. 数字表示法与编码基础

## 📌 定点数（Fixed Point）

- **定义：** 定点数是指小数点的位置是固定的数。广泛用于嵌入式系统、数字电路中。
- 分为：
  - **定点整数：** 不含小数，使用全部位表示整数
  - **定点小数：** 固定小数位数，例如将某些位保留用于小数部分
- 相比浮点数，运算速度快、硬件开销小，但精度和范围较受限。

## 🔢 原码（Sign-Magnitude）

- **定义：** 用一个符号位 + 数值绝对值表示
- 最高位为符号：`0` 表示正，`1` 表示负
- 示例：`+5 → 00000101`，`−5 → 10000101`
- 优点：形式简单
- 缺点：运算复杂，需判断符号

## 🔁 反码（One’s Complement）

- **定义：** 正数的反码就是其本身；负数的反码是在原码基础上**符号位不变，其余位取反**
- 示例：`+5 → 00000101`，`−5 → 11111010`
- 缺点：存在两种表示0的方式（正零和负零）

## ➕ 补码（Two’s Complement）

- **定义：** 负数的补码 = 正数对应二进制取反 + 1
- 示例：`+5 → 00000101`，`−5 → 11111011`
- 特点：
  - 统一了加减法操作，可直接进行运算
  - 只有一种零
  - 最常见的有符号数表示方式！



​	注意模4补码（双符号补码），存储时只需要存储一位符号，运算时自动添加一位

## ⏩ 移码（Excess-K Code）

- **定义：** 在原数值基础上加一个常量偏移量 K，常见为 $K=2^{n-1}$
- 用于表示浮点数的指数部分
- 示例：对偏移值 K=127，指数为 `−3` 表示为 `124`，即 `01111100`

# 二.  加法器

## 🔧 一位全加器结构（Full Adder）

| 输入        | 描述           |
| ----------- | -------------- |
| $AiA_i$     | 被加数的本位   |
| $BiB_i$     | 加数的本位     |
| $CiC_{i-1}$ | 来自低位的进位 |

| 输出     | 描述         |
| -------- | ------------ |
| $Si S_i$ | 本位和       |
| $CiC_i$  | 向高位的进位 |

## 🔗 n位加法器的实现

通过将多个一位全加器串联，可构成一个 n-bit 加法器。

text

```
A = 00000111
B = 00001100
S = 00010011
```

- 本质上是一种**并行输入，串行进位**的结构
- 运算速度受限于进位传播的延迟，因此位数增加时速度下降

## 🚦 加法器的分类对比

| 加法器类型           | 特点                     | 运算速度 |
| -------------------- | ------------------------ | -------- |
| 串行进位的并行加法器 | 每一级进位依赖前一级进位 | 慢       |
| 并行进位的并行加法器 | 所有进位信息同时产生     | 快       |

# 三.  判断溢出

## 方法一：符号位比较法

原理：当两个加数同号，但结果与它们符号不同时，说明补码加法超出表示范围，发生溢出。

- 定义
  - A 的符号位：AS
  - B 的符号位：BS
  - 结果的符号位：SS
- 溢出条件
  - A、B 都为正且结果为负：¬AS ∧ ¬BS ∧ SS
  - A、B 都为负且结果为正：AS ∧ BS ∧ ¬SS
- 逻辑表达式

```
V = (¬AS ∧ ¬BS ∧ SS) ∨ (AS ∧ BS ∧ ¬SS)
```

- V = 1 表示有溢出；V = 0 表示无溢出。

## 方法二：符号位进位比较法

原理：补码加法中，进到符号位的进位与符号位运算后的进位不同，说明跨越了可表示范围。

- 定义
  - CS：加法运算进到符号位（最高位前）的进位
  - Cₙ：符号位运算后产生的进位
- 溢出条件
  - CS ≠ Cₙ（进位不一致）
- 逻辑表达式

```
V = CS ⊕ Cₙ
```

- V = 1 表示有溢出；V = 0 表示无溢出。

## 方法三：双符号位扩展法

原理：对所有操作数及结果都用两位符号扩展（正数“00”，负数“11”）。加／减后若两位符号不同即溢出。

- 符号位编码
  - 正数：00
  - 负数：11
- 结果符号位：SS₁ SS₂
- 溢出条件
  - SS₁ ≠ SS₂
- 逻辑表达式

```
V = SS₁ ⊕ SS₂
```

- 举例
  - A+C： 00,0001111 + 00,1111100 = 01,0001011 → SS₁SS₂=01 → 溢出
  - B−C： 11,1101000 + 11,0000100 = 10,1101100 → SS₁SS₂=10 → 溢出
  - A+B： 00,0001111 + 11,1101000 = 11,1110111 → SS₁SS₂=11 → 无溢出
  - A−B： 00,0001111 + 00,0011000 = 00,0100111 → SS₁SS₂=00 → 无溢出

## 三种方法对比

| 方法           | 标志位     | 核心思想                   | 溢出判断                   |
| -------------- | ---------- | -------------------------- | -------------------------- |
| 符号位比较法   | AS, BS, SS | 同号异结果则溢出           | V=(¬AS∧¬BS∧SS)∨(AS∧BS∧¬SS) |
| 进位判断法     | CS, Cₙ     | 符号位前后进位不一致则溢出 | V=CS⊕Cₙ                    |
| 双符号位扩展法 | SS₁, SS₂   | 结果符号两位不一样则溢出   | V=SS₁⊕SS₂                  |

## 拓展阅读

- MIPS／ARM CPU 和 x86 中的溢出标志（OF）即采用“方法二”来设置

## 🚀 什么是上溢出（Overflow）

**定义**：当浮点数的**指数部分**超过了 IEEE 754 所能表示的最大值时，就会发生上溢出。

### 🧠 触发场景：

- **指数太大**：比如两个非常大的数相乘。
- **累加导致溢出**：如在循环中不断累加一个大数。

### 🧮 IEEE 754 单精度 float 的最大值：

- 最大阶码：`254`（偏移量127，实际指数为 +127）
- 最大尾数：接近 `2 - 2^{-23}`
- 所以最大值约为：**3.4028235 × 10³⁸**

### 💥 举例：

```C
float x = 1e38;
float y = 10.0;
float z = x * y; // 上溢出，结果为 +∞
```

## 🧊 什么是下溢出（Underflow）

**定义**：当浮点数的**指数部分**小于 IEEE 754 所能表示的最小值时，就会发生下溢出。

### 🧠 触发场景：

- **指数太小**
- **两个非常小的数相乘**
- **减法导致精度丢失**：如两个非常接近的数相减

### 🧮 IEEE 754 单精度 float 的最小正值：

- 最小阶码：`1`（偏移量127，实际指数为 -126）
- 最小规格化值：**1.175494 × 10⁻³⁸**
- 更小的值会进入**非规格化区间**（denormalized），最小非规格化值约为：**1.4 × 10⁻⁴⁵**
