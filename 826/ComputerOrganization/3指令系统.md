## 2. 指令格式

### 2.1 变长指令字 vs 定长指令字

定长指令字

-   在指令字最高位部分分配固定若干位用于表示操作码，有 利于简化计算机硬件设计，提高指令译码和识别速度

变长指令字

-   这种方法在不增加指令字长的情况下，可表示更多的指令, 但增加了译码和分析难度，要求更多的硬件支持

## 3.寻址方式

寻址方式是指寻找指令或操作数有效地址（Effective Address, EA）的方式。

### 3.1 指令寻址相关寄存器

在指令执行过程中，有三个关键的寄存器负责确定地址：
*   **PC (Program Counter, 程序计数器)**：存放下一条指令的地址。
*   **BR (Base Register, 基址寄存器)**：用于基址寻址，存放基准地址（通常由操作系统管理）。
*   **IX / IR (Index Register, 变址寄存器)**：用于变址寻址，存放地址偏移量。
*   **IP (Instruction Pointer, 指令指针)**：在 x86 架构中等同于 PC。



### 3.2 常见数据寻址方式对比

| 寻址方式       | 有效地址 (EA) 计算    | 特点与应用场景                              |
| :------------- | :-------------------- | :------------------------------------------ |
| **立即数寻址** | 操作数就在指令中      | 速度最快；常数赋值；不支持修改              |
| **直接寻址**   | $EA = A$ (指令地址码) | 访问内存一次；地址固定，不利于程序浮动      |
| **寄存器寻址** | $EA = R_i$ (寄存器名) | 速度极快；不访问内存；寄存器资源有限        |
| **寄存器间址** | $EA = (R_i)$          | 寄存器存的是内存指针；比直接寻址更灵活      |
| **间接寻址**   | $EA = (A)$            | 扩大寻址范围；需要多次访存（速度慢）        |
| **变址寻址**   | $EA = (IX) + A$       | **IX由用户修改**；常用于数组遍历、循环      |
| **基址寻址**   | $EA = (BR) + A$       | **BR由OS修改**；利于多道程序浮动、扩充空间  |
| **相对寻址**   | $EA = (PC) + A$       | A是偏移量；常用于**转移指令**；利于代码浮动 |
| **堆栈寻址**   | $EA = (SP)$           | 使用堆栈指针 $SP$；常用于函数调用、保护现场 |

---

### 3.3 核心寻址逻辑简图

1.   基址寻址 vs 变址寻址

*   **基址寻址**：$EA = \text{基址寄存器内容(不变)} + \text{指令地址码(变)}$。主要解决程序的逻辑地址到物理地址的转换。
*   **变址寻址**：$EA = \text{变址寄存器内容(变)} + \text{指令地址码(不变)}$。主要解决数组元素定位。

2. 相对寻址

*   公式：$EA = (PC) + A$
*   注意：执行时 $PC$ 已经自动完成 $+1$（或加指令字长），所以 $A$ 是相对于下一条指令的偏移。

3.   间接寻址 (Multi-level Addressing)

*   一级间址：指令 $\rightarrow$ 内存(存放地址) $\rightarrow$ 内存(存放操作数)。
*   符号表示：`MOV AX, [1000H]` (直接) vs `MOV AX, [[1000H]]` (间接)。

## 4. CISC 和 RISC

| **特性**       | **CISC (Complex Instruction Set Computer)** | **RISC (Reduced Instruction Set Computer)** |
| -------------- | ------------------------------------------- | ------------------------------------------- |
| 指令长度       | 可变长度（节省存储空间）                    | 固定长度（通常为 32 或 64 位）              |
| 执行时间       | 多周期（不同指令执行时间差异大）            | 单周期（大多数指令在一个时钟周期内完成）    |
| **寄存器数量** | 较少（更多依赖于内存操作）                  | 较多（倾向于寄存器到寄存器的操作）          |
| 内存访问方式   | 复杂，支持多种寻址模式                      | 简单，仅通过 Load/Store 指令访问内存        |
| **流水线化**   | 难以实现高效流水线                          | 易于实现高度并行的流水线                    |
| **编译器要求** | 较低（指令功能强，代码量小）                | 较高（需要优化指令序列以提高效率）          |
| **硬件复杂度** | 高（微代码控制器复杂）                      | 低（硬件逻辑简单，晶体管利用率高）          |
| 典型代表       | x86 (Intel, AMD)                            | ARM, RISC-V, MIPS, Apple M 系列             |



## 5. RISC-V指令

寄存器组主要包括通用寄存器（General Purpose Registers）和控制状态寄存器（Control and Status Registers）

-   32位架构 (`RV32I`) 32个32位的通用寄存器，64位架构 (`RV64I`)32个64位的通用寄存器 
-   嵌入式架构`RV32E`有16个32位的通用寄存器 
-   支持单精度浮点数（F），或者双精度浮点数（D），另外增加一组独立的通用浮点寄存器组，$f_0~f_{31}$
