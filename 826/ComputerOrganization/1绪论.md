## 1. 指令和指令系统

-   计算机系统由硬件和软件两大部分组成
-   指令是计算机运行的最小的功能单元
-   一台计算机提供的全部指令构成该计算机的指令系统

## 2. 评价计算机性能的指标

### 2.1 常用衡量指标

-   吞吐率（Throughput）：单位时间内完成的任务数量
-   响应时间（Response Time）：从任务提交到结果返回的时间
-   时钟周期 (clock cycle,tick,clock tick,clock): CPU的主脉冲信号，其宽度称为时钟周期。
    -   CPI（Cycles Per Instruction）：平均每条指令所需的时钟周期数
    -   主频（Clock Frequency）：每秒时钟周期数，单位 Hz/GHz，$主频 = 1/时钟周期$

-   MIPS（Million Instructions Per Second）：每秒执行百万条指令，适合粗略比较，但受指令集差异影响大



### 2.2 CPU 时间（CPU Time）

-   计算式：$CPU\ Time = 指令数 \times CPI \times 时钟周期$
-   时钟周期与主频互为倒数，主频越高不一定代表更快，还要看指令数和 CPI



### 2.3 指令系统分类

-   复杂指令集（CISC：Complex Instruction Set Computing）
	-   指令数目多且复杂，指令长度不等，硬件实现更复杂
	-   例子：x86 指令集
	
-   精简指令集（RISC：Reduced Instruction Set Computing）
	-   指令数目与寻址方式精简，指令长度规整，利于流水线与并行
	-   例子：MIPS、RISC-V、ARM 指令集

## 5. 程序执行过程

1.   预处理

展开宏、头文件，删除注释

>   gcc -E src/hello.c -o src/hello.i 

2.   编译

将预处理后的代码翻译成**汇编语言**

>   gcc -S src/hello.i -o src/hello.s

3.   汇编

将汇编代码转换成**机器指令**

>   gcc -c src/hello.s -o src/hello.o

4.   链接

`-l` 用于告诉编译器你需要链接的具体库

`-L` 用于告诉编译器，如果系统默认路径找不到库，应该去哪个目录下搜寻

>   gcc src/hello.o -lc -o src/hello.c



去 `/usr/lib` 下执行 `ls`，会看到两种主要类型：

-   **动态链接库 (`.so`)**：文件名形如 `libc.so.6`。程序运行时才加载，节省空间。
-   **静态链接库 (`.a`)**：文件名形如 `libc.a`。编译时直接打包进可执行文件。
