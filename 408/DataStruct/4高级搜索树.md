## 1.B树

### 2.1 B树基础

#### 2.1.1 外部节点

-   数值为NULL的节点

-   计算**B树的高度**（h）需要计算外部节点

#### 2.1.2 内部节点

| **属性**                | **下限 (Minimum)**                           | **上限 (Maximum)** | **备注**                       |
| ----------------------- | -------------------------------------------- | ------------------ | ------------------------------ |
| **分支数量 (Children)** | $\lceil b/2 \rceil$ （只有根节点可以不满足） | $b$                | 保证树的填充率不低于 50%       |
| **关键字数量 (Keys)**   | $\lceil b/2 \rceil - 1$                      | $b - 1$            | 每个节点包含的关键字比分支少 1 |

#### 2.1.3 高度和节点的关系

设$n$为关键字总数，$m$阶B树

$$
\log_m (n+1) \le h \le \log_{\lceil m/2 \rceil} (\frac{n+1}{2}) + 1
$$

#### 2.1.4 其他

>   注意IO操作不需要最后搜索外部节点，所以算出了高度后需要-1

>   m阶B树，是指分支数量最多为m，这样更好记忆 

>   也可以叫b阶B树为$(\lceil b/2 \rceil,b)$树

### 2.2 B树的插入

-   **核心机制**：插入具体的位置后，看看会不会超过限制，会的话就分裂一下；分裂时将**中间**（或中间靠右）的节点提上去。
-   **特性**：可以一直向上递归提上去，直到根节点分裂，这是 B 树增高的唯一途径。

#### 📌 插入与递归分裂案例演示

**初始状态**：这是一棵完美的 3 阶 B 树，注意**所有叶子节点都在同一层（第三层）**。此时左边叶子 `[12, 15]` 已满。

Plaintext

```
                 [20]             <-- 第一层（根节点）
                /    \
             [10]    [30]         <-- 第二层（内部节点）
            /   \    /   \
         [5] [12,15][25] [35]     <-- 第三层（叶子节点，绝对对齐）
```

操作：插入 18

```
                 [20]              
                /    \
            [10, 15]  [30]         <-- 15 被提上来，该节点变胖
           /   |   \  /   \
         [5] [12] [18][25] [35]    <-- 分裂出新的分支，但仍在同一水平面
```

### 2.3 B树的删除

-   **核心机制**：从叶子删除后，若节点关键字数 $< 1$（即变为 0 个关键字），则：
    1.  优先从左兄弟或右兄弟借一个关键字（需要父节点下移一个，兄弟上移一个）。
    2.  若兄弟也刚好只有 1 个无法借，则进行合并：将该节点与兄弟及父节点中的一个关键字合并成一个新节点，并删除父节点中的那个关键字（可能递归导致父节点关键字数 $< 1$，继续处理）。
-   **非叶子节点转化**：先将自己的后继（或前驱）节点提上来，然后交换一下，这样就可以保证自己**删除的永远都是叶子节点**。

#### 2.3.1 情况#1：已到树根，全树可能降低一层

-   **场景**：当底层的合并操作一路向上递归，最终使得**根节点**的关键字被抽空时触发。
-   **演示**：m = 3

```
【连锁合并导致中间层变空】
           [20]          <-- 根节点
          /    \
        []      [30]     <-- 中间层变空，且右兄弟 30 无法借位
       /
   [10, 15]

【操作】老根节点 20 下落，与子节点全家合并。老树根消失。
         []              <-- 老树根作废
         |
  [10, 15, 20, 30]       <-- 唯一的子节点直接上位成为新树根（树高降低 1 层）
```

>   *(注：合并出的超大节点会紧接着自动触发插入的“自底向上分裂”机制来恢复 3 阶限制。)*

#### 2.3.2 情况#2：通过右旋，向左兄弟借关键码

-   **场景**：当前节点空了，但**左兄弟**很富裕（关键字数 $> 1$）。
-   **操作**：父节点下落，左兄弟最大值上移。
-   **演示**：

```
【初始状态】准备删除 25
       [10]
      /    \
  [5, 8]   [25]  <-- 删掉 25 后变空

【调整：右旋】父节点 10 掉下来，左兄弟的 8 顶上去
       [8]
      /   \
   [5]     [10] 
```

#### 2.3.3 情况#3：通过左旋，向右兄弟借关键码

-   **场景**：当前节点空了，左边没得借，但**右兄弟**很富裕（关键字数 $> 1$）。
-   **操作**：父节点下落，右兄弟最小值上移。
-   **演示**：

```
【初始状态】准备删除 5
       [10]
      /    \
   [5]     [15, 20] <-- 删掉 5 后变空

【调整：左旋】父节点 10 掉下来，右兄弟的 15 顶上去
         [15]
        /    \
     [10]    [20]
```

#### 2.3.4 情况#4：合并（兄弟都不够借）

-   **场景**：当前节点空了，且左右兄弟都处于低保状态（刚好只有 1 个关键字），无码可借。
-   **操作**：把父节点拉下来，和当前节点、兄弟节点一起融合成一个大节点。
-   **演示**：

```
【初始状态】准备删除 20
       [10]
      /    \
   [5]     [20]  <-- 删掉 20，左兄弟 5 不够借

【调整：合并】父节点的 10 被拽下来，与左兄弟 5 彻底融合
   [5, 10]       <-- 融合为一个节点，父节点失去 10
```

>   **连锁反应注意**：情况#4 发生后，父节点失去了一个关键字。如果父节点原先也只有 1 个关键字，此时就会变空，必须把父节点当成新的处理对象，继续向上层执行这 4 种情况的判断。



## 2. 红黑树

-   树根必为黑色 

-   外部节点均为黑色 

-   红节点的孩子必为黑色 —— 不红红

-   从任一节点x到其每个后代nullptr，沿途经过的（真）黑后代一样多 —— 黑高等

    这个黑后代的数目（不计x自身），称作x的<span style="color:#FF0000;">黑高度</span>

>   注意**邓书**中高级搜索树高度的计算：BST不计算外部节点，B树需要计算外部节点，红黑树模糊，书上和ppt不一样

#### 3.2.1 特殊情况

![](/home/ping/Second_Attempt/408/DataStruct/assets/82b3fc7d-c900-43a3-8bce-a9ca84534f6d.png)

| 情况   | 判定                                 | 染色次数 | 旋转次数 | 结果                             |
| ------ | ------------------------------------ | -------- | -------- | -------------------------------- |
| **1**  | 兄弟 `s` 为黑，且 `s` 至少一个红孩子 | 3        | 1~2      | 一蹴而就                         |
| **2R** | `s` 为黑，两孩均黑，`x p` 为红       | 2        | 0        | 一蹴而就                         |
| **2B** | `s` 为黑，两孩均黑，`x p` 为黑       | 1        | 0        | 可能传播 `O(log n)` 层           |
| **3**  | 兄弟 `s` 为红（孩子均黑）            | 2        | 1        | 必随即转入 **BB-1** 或 **BB-2R** |

可能的转换：

-   2B -> 2R
    2R -> 1
    3 -> 1 或 2R

不可能的转换：

-   2B -> 1
    2B -> 3
