## 1. 存储系统基本概念

### 1.1 存储系统的层次结构

- **层次思想**：上一层的存储器作为低一层存储器的高速缓存，上一层的内容是下一层内容的一部分。
- **Cache —— 主存层**
  - 解决 CPU 和主存速度不匹配的问题。
  - 数据调度由硬件自动完成。
  - 对所有程序员透明。
- **主存 —— 辅存层**
  - 解决存储系统容量的问题。
  - 数据调度由硬件和操作系统共同完成（换入换出技术）。
  - 对应用程序员透明。
  - 逐渐发展形成虚拟存储系统。
- 主存与 CPU、Cache、辅存都能交换信息。
- Cache 和主存能与 CPU 直接交换信息。
- 辅存要通过主存与 CPU 交换信息。

### 1.2 存储器的分类

#### 1.2.1 按在计算机中的作用分类

- **主存储器（主存/内存）**
  - 存放计算机运行期间所需的程序和数据。
  - 容量较小，存取速度较快，价格较高。
- **辅助存储器（辅存/外存）**
  - 存放当前暂时不用的程序和数据以及需要永久性保存的信息。
  - 容量大，存取速度较慢，单位成本低。
- **高速缓存存储器（Cache）**
  - 位于主存和 CPU 之间，存放当前 CPU 经常使用的指令和数据。
  - 现代计算机通常将其制作在 CPU 内。
  - 存取速度可与 CPU 速度相匹配，存储容量小，价格高。

#### 1.2.2 按存储介质分类

- **磁表面存储器**：磁盘，磁带。
- **磁芯存储器**。
- **半导体存储器**：MOS 型存储器，双极型存储器。
- **光存储器**：光盘。

#### 1.2.3 按存取方式分类

- **随机存储器（RAM）**
  - 任何一个存储单元都可以随机存取，存取时间与物理位置无关。
  - 主要用于主存或高速缓冲存储器。
  - RAM 分为静态 RAM（SRAM）和动态 RAM（DRAM）。
  - RAM 主要为用户编程设置的。
- **只读存储器（ROM）**
  - 内容只能随机读出而不能写入，信息一旦写入就不变，断电后不消失。
  - 通常用于存放固定不变的程序、常数和汉字字库。
  - ROM 与 RAM 一起统一构成主存的地址域。
  - ROM 和 RAM 的存取方式均为随机存取。
  - 广义上的 ROM 现在可通过电擦除进行写入，写入速度比读取速度慢。
- **串行访问存储器**
  - 对存储单元进行读写操作时，需按其物理位置的先后顺序寻址。
  - 顺序存取存储器（SAM）：磁带，只能按某种顺序存取。
  - 直接存取存储器（DAM）：磁盘、光盘，既不是随机存取，也不是顺序存取。
- **相联存储器（CAM）**
  - 按内容访问，如快表。

#### 1.2.4 按信息的可保存性分类

- **易失性存储器**：断电后信息消失（RAM，主存，Cache）。
- **非易失性存储器**：断电后信息仍保存（ROM，磁表面存储器，光存储器）。
- **破坏性读出**：信息读出后原存储信息被破坏（DRAM 芯片，读出后需重写）。
- **非破坏性读出**：信息读出后原存储信息不被破坏（SRAM 芯片，磁盘，光盘）。

| 存储单元类型 | 存储器类型 | 是否可读写 | 数据保存性 | 主要作用 | 常见实现技术 |
| ------------ | ---------- | ---------- | ---------- | -------- | ------------ |
| 缓存（Cache） | SRAM | 可读写 | 断电丢失 | 提高 CPU 访问速度，存放近期使用数据 | 高速静态随机存取存储器 |
| 主存（内存） | DRAM | 可读写 | 断电丢失 | 存储运行中的程序和数据 | 动态随机存取存储器 |
| 硬盘驱动器（HDD） | 非易失性介质 | 可读写 | 断电保存 | 长期存储数据和程序 | 磁性存储（旋转磁盘） |
| 固态硬盘（SSD） | 闪存（Flash） | 可读写 | 断电保存 | 快速读写，取代传统硬盘用于长期存储 | NAND 闪存 |
| 只读存储器（ROM） | ROM | 仅可读 | 断电保存 | 存储固件，如 BIOS 启动信息 | 掩膜 ROM、EPROM、EEPROM 等 |
| 寄存器 | SRAM 或专用电路 | 可读写 | 断电丢失 | CPU 中暂存数据或指令，访问速度最快 | 高速逻辑电路或 SRAM |
| 显存（GPU RAM） | GDDR/DDR RAM | 可读写 | 断电丢失 | 为图形处理器提供高速图像数据存储 | 图形双倍数据率 RAM |
| 虚拟内存 | 由磁盘模拟 RAM | 可读写 | 断电保存 | 扩展内存空间，提升多任务运行能力 | 磁盘空间（如分页文件） |

### 1.3 存储器的性能指标

#### 1.3.1 存储容量

- 存储容量 = 存储字数 × 字长（如 1M × 8 bit）。
- 存储字数表示存储器的地址空间大小（由 MAR 决定）。
- 字长表示一次存取操作的数据量（由 MDR 决定）。

#### 1.3.2 单位成本

- 每位价格 = 总成本 / 总容量。

#### 1.3.3 存取速度

- **数据传输率** = 数据宽度 / 存取周期。
- 数据传输率中的 K、M 是 10 的次方，只有存储容量是 2 的次方。
- **存取时间（$$ T_a $$）**：从启动一次存储器到完成该操作所经历的时间。
- **存取周期（$$ T_m $$）**：存储器进行一次完整的读写操作所需的全部时间，$$ T_m = T_a + 恢复时间 $$。
- **主存带宽（$$ B_m $$）**：数据传输率，即每秒从主存进出信息的最大数量（单位字/秒）。


## 2. 主存储器

### 2.1 主存储器的基本组成

#### 2.1.1 基本元件

- MOS 管：作为通电"开关"。
- 电容：存储电荷（即存储二进制 0/1）。

#### 2.1.2 存储芯片的结构

- **译码驱动电路**：译码器将地址信号转化为字选通线的高低电平。
- **存储矩阵（存储体）**：由多个存储单元构成，每个存储单元由多个存储元构成。
- **读写电路**：每次读/写一个存储字。
- **读（RD）/写（WR）控制线**：决定芯片进行读还是写操作（可能分开两根，也可能只有一根）。
- **片选线（CS）**：确定哪个存储芯片被选中，可用于容量扩充。
- **引脚最低数目**：片选线（1）+ 控制线（2）+ 数据线 + 地址线。

**地址复用技术（DRAM）**
- 由于 DRAM 芯片容量大、地址位数多，为减少地址引脚线，采用地址复用技术。
- DRAM 分两次发送地址（行地址和列地址），地址线数量减半。
- 引脚数 = 地址线减半 + 数据线 + 行通选（1）+ 列通选（1）+ 读写控制线（2）。
- 片选线用行通选线替代。

**总容量** = 存储单元个数 × 存储字长。
常见描述：8K × 8 位，即 $$ 2^{13} \times 8 $$ bit = 8 KB。

#### 2.1.3 寻址方式

- 题目未明确指定按字编址时，默认按字节编址（1 字节 = 8 位）。
- 32 位计算机：32 bit = 4 Byte = 1 Word。
- 64 位计算机：64 bit = 8 Byte = 1 Word。

| 术语 | 描述 |
| ---- | ---- |
| 字存储单元 | 存放一个机器字的存储单元 |
| 字地址 | 对字存储单元的地址 |
| 字节存储单元 | 存放一个字节的存储单元 |
| 字节地址 | 对字节存储单元的地址 |
| 按字寻址计算机 | 可编程最小单位是字存储单元，地址单位为"字" |
| 按字节寻址计算机 | 可编程最小单位是字节存储单元，地址单位为"字节" |
| 机器字包含的字节数 | 一个机器字通常包含多个字节（如 32 位机器字 = 4 字节） |
| 字存储单元的字节地址 | 一个字存储单元中可包含多个字节地址（每个字节可单独寻址） |

#### 2.1.4 主存储器的组成部分

- 数据线的宽度 = MDR 的宽度 = 存储字长。
- 地址线的宽度 = MAR 的宽度 = 存储字数。

### 2.2 DRAM 和 SRAM

#### 2.2.1 比较

| | DRAM | SRAM |
| ---- | ---- | ---- |
| 主要用途 | 主机内存 | 高速缓存 |
| 存储信息 | 电容 | 双稳态触发器 |
| 破坏性读出 | 是 | 否（读出后仍保持原状态，不需要再生） |
| 需要刷新 | 需要 | 不需要 |
| 送行列地址 | 分两次送 | 同时送 |
| 运行速度 | 慢 | 快 |
| 集成度 | 高 | 低 |
| 存储成本 | 低 | 高 |

#### 2.2.2 SDRAM

- **同步时钟控制**：所有操作与系统时钟同步，避免异步 DRAM 的时序瓶颈。
- **流水线操作**：可在处理当前指令的同时接收下一条指令，提高指令吞吐率。
- **多 Bank 结构**：内部分为多个存储块，支持并行访问，进一步提升性能。
- **刷新机制**：作为 DRAM 的一种，SDRAM 仍需定期刷新以维持电容中的数据。

#### 2.2.3 DRAM 的刷新

**刷新概念**
- DRAM 电容电荷维持时间短，即使电源不断电，信息也会自动消失，因此每隔一段时间必须刷新（一般取 2 ms，称为刷新周期或再生周期）。
- DRAM 的刷新是以**行**为单位的。
- 一次完整的刷新过程只需要**占用一个存储周期**。
- DRAM 的存取时间比恢复时间短。

##### （1）集中刷新

- 在规定的一个刷新周期内，对全部存储单元集中一段时间逐行进行刷新，此刻必须停止读/写操作。
- 刷新过程：用 $$ 0.5 \mu s \times 128 = 64 \mu s $$ 的时间对 128 行进行逐行刷新，这 64 μs 内不能进行读/写操作，称为"死时间"或"访存死区"。
- 由于存取周期为 0.5 μs，刷新周期为 2 ms，即 4000 个存取周期。
- 刷新与存取不能并行，因为内存只有一套地址译码和片选装置，刷新与存取有相似的过程，需要选中一行（片选线、地址线、地址译码器被占用）；同理，刷新操作之间也不能并行（一次只能刷一行）。

##### （2）分散刷新

- 对每行存储单元的刷新分散到每个存取周期内完成。
- 将机器的存取周期 $$ t_c $$ 分成两段：前半段 $$ t_s $$ 用于读/写或维持信息，后半段 $$ t_R $$ 用于刷新。
- 刷新过程：每个存取操作后绑定一个刷新操作，存取周期变为 0.5 μs + 0.5 μs = 1 μs。
- 每有 128 个读取操作，就会把 0~127 行全部刷新一遍，故每隔 128 μs 可将存储芯片全部刷新一遍（刷新周期为 1 μs × 128 = 128 μs），远短于 2 ms。
- 优点是不存在停止读/写的死时间；缺点是存取周期加长，整个系统速度降低，且刷新过于频繁造成浪费。

##### （3）异步刷新

- 既缩短"死时间"（仍然存在），又充分利用最大刷新间隔为 2 ms 的特点。
- 刷新过程：在 2 ms 内对 128 行各刷新一遍，即每隔 $$ 2000 \mu s / 128 \approx 15.6 \mu s $$ 刷新一行，每行刷新的时间仍为 0.5 μs。
- 刷新一行只需停止一个存取周期，但每行的刷新间隔时间仍为 2 ms，死时间为 0.5 μs。
- 特点：将 DRAM 的刷新安排在 CPU 对指令的译码阶段（此阶段 CPU 不访问存储器），既克服了分散刷新需独占 0.5 μs 使存取周期加长、降低系统速度的缺点，又不会出现集中刷新的访存"死区"问题，从根本上提高了整机工作效率。

### 2.3 只读存储器 ROM

#### 2.3.1 ROM 的特点

- 结构简单，位密度比可读写存储器高。
- 具有非易失性，可靠性高。

#### 2.3.2 ROM 的类型

| 类型 | 编程方式 | 是否可重写 | 擦除方式 | 特点与应用场景 |
| ---- | ---- | ---- | ---- | ---- |
| MROM | 生产时写入 | 不可重写 | 不可擦除 | 可靠性高、成本低、灵活性差，适用于固定程序如嵌入式系统固化代码 |
| PROM | 用户一次性编程 | 不可重写 | 不可擦除 | 一次性写入后不可更改，适用于量产前的程序定型 |
| EPROM | 用户多次编程 | 可重写（有限） | 紫外线擦除 | 可多次编程但写入慢，需物理操作擦除，适用于开发阶段或低频更新场景 |
| Flash | 在线编程 | 可重写 | 电擦除（块级） | 写入前需擦除，写慢读快，广泛用于 U 盘、嵌入式设备、手机存储等 |
| SSD | 控制器管理编程 | 可重写 | 电擦除（页/块） | 基于 Flash，读写快、低功耗、非易失性，用于高性能存储如笔记本、服务器硬盘 |

### 2.4 多模块存储器

#### 2.4.1 概念

- 一种**空间并行技术**，利用多个结构完全相同的存储模块的并行工作来提高存储器的吞吐率。

#### 2.4.2 单体并行存储器

- **定义**：存储器中只有一个存储体，每个存储单元存储 m 个字，总线宽也为 m 个字，地址必须顺序排列并处于同一存储单元。
- **过程**：在一个存取周期内，从同一地址取出 m 条指令，然后逐条送至 CPU。
- **缺点**：指令和数据在主存内必须连续存放，一旦遇到转移指令或操作数不能连续存放，效果不明显。

#### 2.4.3 多体并行存储器

由多体模块组成，每块都有相同容量和读取速度，各模块都有独立的读写控制电路、MAR 和 MDR，既能并行工作也能交叉工作。

**（1）高位交叉编址（顺序方式）**

- **特点**：先在一个模块内访问，访问完后再转到下一个模块。
- **编号**：高位地址表示体号（模块号），低位地址表示体内地址。
- **优点**：某个模块故障时其他模块可照常工作；通过增添模块扩充容量方便。
- **缺点**：各模块串行工作，存储器带宽受限，不能提高吞吐量。

**（2）低位交叉编址（交叉方式）**
- **特点**：连续地址分布在相邻的不同模块内，同一模块内的地址不连续，满足程序的局部性原理。

- **编号**：高位地址表示体内地址，低位地址表示体号（模块号）。

- **优点**：对连续字的成块传送可实现多模块并行存取，提高存储器带宽。

- **计算**：每个模块按"模 m"交叉编址，$$ 模块号 = 单元地址 \bmod m $$。

- **启动方式**
  - **轮流启动**：设模块字长 = 数据总线宽度，模块存取周期为 T，总线周期（每次传输时间）为 r。存储器交叉模块数最小为 $$ m = T / r $$。每隔 $$ 1/m $$ 个存取周期轮流启动各模块，则每隔 $$ 1/m $$ 个存取周期可读出或写入一个数据，存取速度提高 m 倍。当模块数 $$ \ge m $$ 时，可保证 T 时间后再启动该模块时上次操作已完成，流水线不断。连续存取 m 个字的时间为 $$ T + (m - 1)r $$。判断访问冲突规则：给定的访存地址在相邻的四次访问中出现在同一个存储模块中（m=4 时）。
  - **同时启动**：所有模块一次并行读/写的总位数等于数据总线位数，同时启动所有模块进行读/写。

### 2.5 主存储器与 CPU 的连接

#### 2.5.1 连接原理

- 主存储器通过地址总线、数据总线和控制总线与 CPU 协同工作，由 MAR 定位数据位置、控制总线指示操作类型，通过 MDR 完成数据在存储矩阵中的读写传输，整体性能受数据总线宽度与频率共同影响。

#### 2.5.2 主存容量的扩展

**（1）位扩展法**

- 例如：8K × 8 位的存储器由 8 片 8K × 1 位的 RAM 组成。
- 地址线并行，数据线一一接上。

**（2）字扩展法**

- **线选法（一对一）**：例如 16K × 8 位的存储器由 2 片 8K × 8 位的存储器组成。地址线为 $$ A_0 \sim A_{12} $$ 共 13 位，译码线为 $$ A_{13} $$ 和 $$ A_{14} $$ 共 2 位，由片选信号区分各芯片地址范围。2 位二进制时只能利用 01、10。
- **译码片选法（n 对 $$ 2^n $$）**：有 4 块芯片时只需 2 条译码线，可利用 00、01、10、11 四种状态。译码器将左边地址线表示的二进制意义映射到右边十进制的选通线。

**（3）字位同时扩展法**

- 先通过多片叠加实现位扩展（如 2 片 4 位芯片扩展为 8 位），等价于一个 8 位存储芯片。
- 再通过译码片选实现字扩展，在不同的地址线中选择不同的芯片组合工作。

### 3.2 固态硬盘 SSD

#### 3.2.1 原理

- 基于**闪存技术（Flash Memory）**，属于电可擦除 ROM（EEPROM）。

#### 3.2.2 组成

- **闪存翻译层**：负责翻译逻辑块号，找到对应页（Page）。
- **存储介质**：多个闪存芯片（Flash Chip），每个芯片包含多个块（Block），每个块包含多个页（Page）。

#### 3.2.3 读写性能特性

- **以页为单位读/写**：相当于磁盘的"扇区"。
- **以块为单位"擦除"**：擦干净的块，其中的每页都可以写一次，读无限次。
- **支持随机访问**：系统给定逻辑地址，闪存翻译层可通过电路迅速定位到对应物理地址。
- **读快，写慢**：要写的页如果有数据，则不能写入，需要将块内其他页全部复制到一个新的（擦除过的）块中，再写入新的页。

#### 3.2.4 与机械硬盘相比的特点

- SSD 读写速度快，随机访问性能高，用电路控制访问位置；机械硬盘通过移动磁臂、旋转磁盘控制访问位置，有寻道时间和旋转延迟。
- SSD 安静无噪音、耐摔抗震、能耗低、造价更高。
- SSD 的一个"块"被擦除次数过多（重复写同一个块）可能会坏掉，而机械硬盘的扇区不会因写入次数过多而损坏。

#### 3.2.5 磨损均衡技术

- 思想：将"擦除"平均分配在各个块上，提升使用寿命。
- **动态磨损均衡**：写入数据时，优先选择累计擦除次数少的新闪存块。
- **静态磨损均衡**：SSD 检测并自动进行数据分配、迁移，让老旧的闪存块承担以读为主的存储任务，让较新的闪存块承担更多的写任务。

## 4. 高速缓存存储器

### 4.1 Cache 的基本原理

#### 4.1.1 基本概念

- 高速缓冲存储器（Cache）位于主存与 CPU 之间，CPU 可直接对其存取数据，减少访问时间，提高系统运行速度。
- CPU 与 Cache或主存 的信息交互单位为**字**，Cache 与主存的信息交互单位为**块**（一个块通常由若干字组成）。
- Cache 利用局部性原理：将程序中正在使用的部分存放在容量较小但速度更快的 Cache 中。

#### 4.1.2 局部性原理

- **时间局部性**：在最近的未来要用到的信息，很可能是现在正在使用的信息（如循环）。
- **空间局部性**：在最近的未来要用到的信息（指令和数据），很可能与现在正在使用的信息在存储空间上是邻近的（如数组按行存储时，先行后列的访问方式空间局部性更好）。

#### 4.1.3 读写过程

1. Cache 从主存中一并读取目标数据及其附近空间的数据（通常以块为单位取出）。
2. CPU 取数据时优先从 Cache 中读取（速度快）。
3. CPU 计算完后将数据再次返回给 Cache。
4. 整个过程全部由**硬件实现**。

#### 4.1.4 性能分析

设 $$ t_c $$ 为访问一次 Cache 所需时间，$$ t_m $$ 为访问一次主存所需时间。

- **Cache 命中率 H**：$$ H = \frac{Cache 命中次数}{Cache 命中次数 + 访问主存次数} $$
- **缺失率 M**：$$ M = 1 - H $$
- **Cache-主存系统的平均访问时间 t**
  - 先访问 Cache，未命中再访问主存：$$ t = H t_c + (1-H)(t_c + t_m) $$
  - 同时访问 Cache 和主存：$$ t = H t_c + (1-H) t_m $$
- **系统平均访问时间** = 命中概率 × 命中时间 + 缺失概率 × 平均访存次数 × 一次总线读突发事务所需时间。
- **性能效率** = $$ \frac{访问 Cache 的时间}{系统平均访问时间} $$

#### 4.1.5 要解决的关键问题

- **地址映射**：主存块如何存放在 Cache 中，如何将主存地址转换为 Cache 地址。
- **替换策略**：Cache 满后，使用何种策略对 Cache 块进行替换或淘汰。
- **写入/更新策略**：如何既保证主存块和 Cache 块的数据一致性，又尽量提升效率。

### 4.2 Cache-主存映射方式

#### 4.2.1 全相联映射（Fully Associative Mapping）

全相联映射最任性，主存块可以塞进 Cache 的**任意一行**。因此，它的地址不需要指出“哪一行”，只需要区分“哪个块”。

##### 1. 地址格式

主存地址被划分为两部分：

$$
\text{主存地址} = \begin{array}{|c|c|} \hline \text{主存字块标记 (Tag)} & \text{块内地址 (Word Offset)} \\ \hline \end{array}
$$

-   块内地址位数：$b = \log_2(\text{主存块大小 / 字节或字})$

##### 2. 映射公式与查找过程

-   映射公式：无固定公式，主存块 $j$ 可映射到 Cache 的任意行 $i$。
-   **查找过程**：一个一个对比，直到找到对应的Tag

#### 4.2.2 直接映射（Direct Mapping）

直接映射最死板，主存块只能映射到固定的 Cache 行。为了快速定位，主存地址中必须包含 **Cache 行号**。

##### 1. 地址格式

主存地址被划分为三部分：

$$
\text{主存地址} = \begin{array}{|c|c|c|} \hline \text{主存字块标记 (Tag)} & \text{Cache 行号 (Index)} & \text{块内地址 (Word Offset)} \\ \hline \end{array}
$$

-   块内地址位数：$b = \log_2(\text{主存块大小})$
-   Cache 行号位数：$c = \log_2(\text{Cache 总行数})$

##### 2. 映射公式与查找过程

-   映射公式：
    $$
    \text{Cache 行号 }  = \text{主存块号} \bmod \text{Cache总行数 } 
    $$

-   **查找过程**：

    1.  CPU 根据地址中的 **Index** 直接定位到固定的那一行。
    2.  **仅比较一次**：将地址中的 **Tag** 与该行的标记进行比较，并检查有效位。
    3.  若相同且有效则命中，否则不命中。

#### 4.2.3 组相联映射（Set-Associative Mapping）

组相联是前两者的折中。Cache 被分为若干个“组”（Set），组间是直接映射（算号对号入座），组内是全相联（随意放）。常见的有“2路组相联”（每组2行）、“4路组相联”（每组4行）。

##### 1. 地址格式

主存地址被划分为三部分：

$$
\text{主存地址} = \begin{array}{|c|c|c|} \hline \text{主存字块标记 (Tag)} & \text{Cache 组号 (Set Index)} & \text{块内地址 (Word Offset)} \\ \hline \end{array}
$$

-   块内地址位数：$b = \log_2(\text{主存块大小})$

-   Cache 组号位数：$s = \log_2(\text{Cache 总组数 } Q)$

    >   *注：$\text{Cache 总组数 } Q = \frac{\text{Cache 总行数}}{\text{路数 (每组行数)}}$*

-   主存字块标记 (Tag) 位数：$t = \text{主存地址总位数} - s - b$

##### 2. 映射公式与查找过程

-   **查找过程**：
    1.  CPU 根据地址中的 **Set Index** 定位到对应的 Cache 组。
    2.  **组内多路并行比较**：将地址中的 **Tag** 与该组内**所有行**（如果是 $k$ 路，就是 $k$ 行）的标记同时进行硬件并行比较。
    3.  若匹配且有效则命中。

| 映射方式 | 组数 S | 相联度 E | 组索引位 s | 特点 |
| ---- | ---- | ---- | ---- | ---- |
| 直接映射 | $$ S = C $$ | $$ E = 1 $$ | $$ s = \log_2 C $$ | 硬件最简单、访问最快、冲突丢失多 |
| 组相联映射 | $$ S = C/E $$ | $$ E = 2, 4, \dots $$ | $$ s = \log_2 (C/E) $$ | 在复杂度与命中率之间折中 |
| 全相联映射 | $$ S = 1 $$ | $$ E = C $$ | $$ s = 0 $$ | 放置最灵活、冲突丢失最少、硬件与替换最复杂 |

### 4.3 Cache 替换算法

#### 4.3.1 随机算法（RAND）

- Cache 满时随机选择一块替换。
- 实现简单，但完全未考虑局部性原理，命中率低，效果不稳定。

#### 4.3.2 先进先出（FIFO）

- 按调入 Cache 的先后顺序淘汰，先进先替换。
- 需要记录进入 Cache 的先后次序，实现简单。
- 未考虑局部性原理，可能有"抖动现象"（刚被替换的块很快又被调入）。

#### 4.3.3 近期最少使用（LRU）

- 替换最近最少使用的块。
- 需要记录访问先后次序，实现复杂且开销大。
- 根据程序访问局部性原理选择，命中率较高。

#### 4.3.4 最不经常使用（LFU）

- 统计使用次数，替换使用次数最少的块。
- 需要硬件计数器支持。

### 4.4 Cache 写策略

#### 4.4.1 全写法 + 非写分配法

**（1）全写法（Write-Through）**

- CPU 对 Cache 写命中时，同时将数据写入 Cache 和主存，一般使用写缓冲（Write Buffer）。
- 访存次数增加，速度变慢，但能保证数据一致性。
- CPU 写入 Cache 的速度远快于写入主存，故需写缓冲暂存，再慢慢写入主存。

**（2）非写分配法（No-Write-Allocate）**

- CPU 对 Cache 写不命中时，只写入主存，不调入 Cache。

#### 4.4.2 回写法 + 写分配法

**（1）回写法（Write-Back）**

- CPU 对 Cache 写命中时，只修改 Cache 内容，不立即写入主存，只有当该块被换出时才写回主存。
- 减少访存次数，但存在数据不一致的隐患。
- 在 Cache 中设置"脏位"（修改位），当一块中任一单元被修改时脏位置 1。
- 替换时若脏位为 1，则必须先写回主存再调入新块；若脏位为 0，则直接覆盖即可。

**（2）写分配法（Write-Allocate）**

- CPU 对 Cache 写不命中时，将主存中的块调入 Cache，再在 Cache 中修改。

| 方法 | 触发条件 | 操作 | 优点 | 缺点 |
| ---- | ---- | ---- | ---- | ---- |
| 全写法（Write-Through） | 写命中 | 同时将数据写入 Cache 与主存，借助写缓冲 | 保证数据一致性 | 访存次数多，写入延迟较高 |
| 回写法（Write-Back） | 写命中 | 仅更新 Cache，设置脏位；块被替换时再写回主存 | 减少主存写次数 | 可能导致主存数据不一致 |
| 写分配法（Write-Allocate） | 写不命中 | 先将对应块调入 Cache，再在 Cache 中执行写操作 | 首次写入后可利用 Cache 加速 | 首次写入需额外调块，延迟增加 |
| 非写分配法（No-Write-Allocate） | 写不命中 | 直接将写操作发往主存，不在 Cache 中调入该块 | 简化写流程，无块调度开销 | 写后无法利用 Cache 加速 |
