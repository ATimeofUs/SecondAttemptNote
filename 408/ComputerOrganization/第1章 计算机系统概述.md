## 2. 计算机系统层次结构

### 2.1 计算机硬件的基本组成

#### 2.1.1 冯诺依曼结构
**特点：**
1. 采用 **"存储程序"** 的工作方式：
   - 将指令以二进制代码的形式事先输入计算机的主存储器，然后按其在存储器中的首地址执行程序的第一条指令，以后就按该程序的规定顺序执行其他指令，直至程序执行结束。
2. 计算机由 **五大部件** 组成：
   - 输入设备、输出设备、存储器、运算器、控制器。
3. 指令和数据以同等地位存储在存储器中，并 **按地址** 寻访。
4. 指令和数据均以 **二进制代码** 表示：
   - CPU 区分指令和数据的依据是指令周期的不同阶段。
   - 数据由指令的地址码给出。
5. **指令 = 操作码 + 地址码**：
   - 操作码表示操作的性质。
   - 地址码表示操作数在存储器中的位置。
6. 指令在存储器内 **按顺序** 存放：
   - 通常顺序执行，特定条件下可根据运算结果或设定条件改变执行顺序。
7. **以运算器为中心**，输入/输出设备通过运算器与存储器传送数据。
8. 是 **单处理机**。
9. 基本工作方式是 **控制流驱动方式**。

#### 2.1.2 现代计算机结构
**特点：**
1. **以存储器为核心**。
2. **计算机** = 主机 + 输入/输出设备。
3. **主机** = CPU + 主存。
4. **CPU** = 运算器 + 控制器：
   - 控制器：指挥各部件使得程序执行。
   - 运算器：计算机的执行部件，进行算术运算和逻辑运算。
5. **存储器** = 主存 + 辅存，存放数据和指令：
   - 主存储器（内存）：CPU 可直接访问，工作方式为按地址存取。
   - 辅助存储器：需调入主存后才可被 CPU 访问。
6. **I/O 设备** = 输入设备 + 输出设备：
   - 输入设备：将信息转换为机器能够识别的形式。
   - 输出设备：将结果转换为人们熟悉的形式。


### 2.2 计算机的功能部件

#### 2.2.1 存储器
**基本组成：**
- **存储体**：存放二进制信息，由许多存储单元组成。
- **地址寄存器（MAR）**：存放访存地址，经地址译码后找到所选存储单元。
  - 用于寻址，位数反映最多可寻址的存储单元个数。
  - 若 MAR 为 10 位，则最多 $$ 2^{10} = 1024 $$ 个存储单元，记为 1 K。
  - MAR 的长度与 PC 的长度相等（两者均存放地址）。
- **数据寄存器（MDR）**：暂存要从存储器中读或写的信息。
  - 位数 = 存储字长。
- **时序控制逻辑**：产生存储器操作所需的各种时序信号。

> 现代计算机中，MAR 和 MDR 存在于 CPU 中，不视为存储器的一部分。

**相关概念：**
- **存储元**：存储二进制的电子元件，每个存储元可存 1 bit。
- **存储单元**：每个存储单元存放一串二进制代码，包含若干存储元。
- **存储字**：存储单元中二进制代码的组合。
  - 存储字长：存储单元中二进制代码的位数。
  - 机器字长：计算机能一次处理的二进制代码长度。
  - 指令字长：指令的二进制长度。

#### 2.2.2 运算器
**相关概念：**
- **ALU**（算术逻辑单元）：运算器的核心，用电路实现各种算术运算、逻辑运算。
- **ACC**（累加器）：存放操作数或运算结果。
- **MQ**（乘商寄存器）：乘/除运算时存放操作数或运算结果。
- **X**：通用操作数寄存器，存放操作数。
- （以下为可选）**IX**（变址寄存器）、**BR**（基址寄存器）、**PSW**（程序状态寄存器）：存放 ALU 运算标志信息或处理机状态信息（如溢出、进位、结果负等）。

#### 2.2.3 控制器
**相关概念：**
- **PC**（程序计数器）：存放下一条指令的地址，并跟踪下一条要执行的指令的地址。
- **IR**（指令寄存器）：存放当前正在执行的指令，内容来自 MDR。
  - 操作码 OP(IR) 送至 CU。
  - 地址码 Ad(IR) 送往 MAR，用以取操作数。
- **CU**（控制单元）：分析指令，给出控制信号。

### 2.3 计算机系统的细节

#### 2.3.1 系统组成
- **CPU**：中央处理器。
- **PC**：程序计数器。
- **MAR**：存储器地址寄存器。
- **ALU**：算术逻辑部件。
- **IR**：指令寄存器。
- **MDR**：存储器数据寄存器。
- **GPRs**：通用寄存器组（早期即累加器）。

#### 2.3.2 指令的执行
完成一条指令的典型步骤：
1. 将第一条指令的地址加载到程序计数器（PC）中。
2. 将 PC 的地址传送到地址寄存器（MAR）。
3. 发送读信号，从内存中读取该地址对应的指令，并存入数据寄存器（MDR）。
4. 将 MDR 中的指令送入指令寄存器（IR）。
5. 对指令进行译码，识别操作类型及所需操作数地址。
6. 根据需要，从寄存器或内存中取操作数。
7. 将操作数送入算术逻辑单元（ALU），执行指令规定的操作。
8. 将运算结果存入目标寄存器或写回内存。
9. 更新程序计数器（PC）以指向下一条指令地址。

## 3. 计算机软件

### 3.1 两类软件
- **系统软件**：管理计算机系统的硬件资源，向上层应用程序提供服务。
  - 如：操作系统、网络服务程序、语言处理程序、数据库管理系统、程序设计语言、分布式软件系统。
  - 注意：**数据库系统不是系统软件**。
- **应用软件**：按应用场景需要编制成的各种程序，直接为用户提供服务。
  - 如：办公软件、多媒体软件、辅助设计软件、企业应用软件、网络应用软件、安全防范软件、娱乐休闲软件。

### 3.2 三种级别的语言
1. **机器语言**：二进制代码，是计算机唯一可以直接识别和执行的语言。
2. **汇编语言**：助记符，需由汇编程序翻译为机器语言后执行。
3. **高级语言**：如 C、C++、Java。
   - 编译方式：高级语言 → 汇编语言 → 机器语言。
   - 解释方式：高级语言 → 机器语言（边翻译边执行，不生成可执行文件）。

**翻译程序**：将高级语言程序转换为机器语言程序。
- **汇编程序（汇编器）**：将汇编语言程序翻译为机器语言程序。
- **解释程序（解释器）**：将源程序中的语句按执行顺序逐条翻译成机器指令并立即执行（每次执行都需翻译）。
- **编译程序（编译器）**：将高级语言程序翻译成汇编语言或机器语言程序（只需翻译一次）。

### 3.3 软件和硬件的逻辑功能等价性
同一个功能既可以用硬件实现（性能高、成本高），也可以用软件实现（性能低、成本低）。
- 例如：乘法运算可用专门硬件电路实现，也可用软件多次加法实现。

### 3.4 指令集体系结构（ISA）
- **软件和硬件之间的界面**。
- 定义计算机支持哪些指令、每条指令的作用及用法。
- ISA 是软件能够感知到的部分，也称 **软件可见部分**。



## 4. 计算机系统的层次结构

- **微程序机器层**：由硬件直接执行微指令。
- **传统机器语言层**：用微程序解释机器指令。
- **操作系统层**：用机器语言解释操作系统。
- **汇编语言层**：用汇编程序翻译成机器语言程序。
- **高级语言层**：用编译程序翻译成汇编语言程序。

> 下层是上层的基础，上层是下层的拓展。  
> 没有配备软件的纯硬件系统称为 **裸机**。  
> 第 3~5 层称为 **虚拟机**（软件实现的机器）。

**比较：**
- **计算机体系结构**：机器语言程序员所见到的计算机系统的属性（概念性结构与功能特性）。
  - 如：指令系统、数据类型、寻址技术、I/O 机理。
  - 关注如何设计硬件与软件之间的接口（如有无乘法指令）。
- **计算机组成原理**：实现计算机体系结构所体现的属性，对程序员"透明"（看不见）。
  - 如：具体指令的实现、如何用硬件实现所定义的接口（如何实现乘法指令）。

### 5.3 指令执行过程的描述（以取数指令为例）
1. **取指令**：PC → MAR → M → MDR → IR（根据 PC 取指令到 IR）。
2. **分析指令**：OP(IR) → CU（指令译码并送出控制信号）。
3. **执行指令**：Ad(IR) → MAR → M → MDR → ACC（取数操作）。


## 6. 计算机的性能指标（重点）

### 6.1 字长
- **机器字长（字长）**：计算机进行一次整数运算所能处理的二进制的位数。
- 字长一般等于 **通用寄存器的位数或 ALU 的宽度**。
- 字长越长，数的表示范围越大，计算精度越高。
- 字长通常选定为字节（8 位）的整数倍。

### 6.2 数据通路带宽
- 外部数据总线一次能并行传送信息的位数（非 CPU 内部数据总线宽度）。
- 各个子系统通过数据总线连接形成的数据传送路径称为 **数据通路**。

### 6.3 主存容量
- 主存储器所能存储信息的最大容量。
- 通常以字节衡量（1 B = 8 bit）。
- 也可用 $$ 字数 \times 字长 $$ 表示。
  - 例：MAR 为 16 位，则 $$ 2^{16} = 65536 $$ 个存储单元（64 K，1 K = 1024）；MDR 为 32 位，则存储容量为 $$ 64K \times 32 $$ 位。

### 6.4 吞吐量和响应时间
- **吞吐量**：系统在单位时间内处理请求的数量。
- **响应时间**：用户发送请求到系统响应并获得所需结果的等待时间（通常包括 CPU 时间和等待时间）。

### 6.5 主频和 CPU 时钟周期
- **主频（CPU 时钟频率）**：机器内部主时钟的频率，代表每秒执行多少个时钟周期数。
  - 值越大，操作所需时间越少，CPU 运行速度越快。
- **CPU 时钟周期**：机器内部主时钟脉冲信号的宽度（节拍脉冲或 T 周期），即 **主频的倒数**。
  - 是 **CPU 工作的最小时间单位**，每个动作至少需要 1 个时钟周期。
  - 公式：$$ 时钟周期 = \frac{1}{主频} $$，如主频为 2.4 GHz，则时钟周期 = $$ \frac{1}{2.4G} $$ 秒。

### 6.6 CPI（Cycles Per Instruction）
- **执行一条指令所需要的时钟周期数**（平均值）。
- CPI 与系统结构、指令集、计算机组织有关，与时钟频率无关。
- 公式：$$ CPI = \frac{时钟周期数量}{指令数量} $$。
- **IPS**（每秒执行多少条指令）：$$ IPS = \frac{主频}{平均CPI} $$。

### 6.7 CPU 执行时间
- 运行一个程序所花费的时间。
- 公式：$$ 执行时间 = 时钟周期数量 \times 时钟周期 = \frac{时钟周期数量}{主频} = \frac{指令条数 \times CPI}{主频} $$。

### 6.8 MIPS / MFLOPS / GFLOPS / TFLOPS / PFLOPS

| 指标   | 全称                                       | 含义                     | 计算公式                                                |
| ------ | ------------------------------------------ | ------------------------ | ------------------------------------------------------- |
| MIPS   | Million Instructions Per Second            | 每秒执行百万条指令       | $$ MIPS = \frac{指令条数}{执行时间 \times 10^6} = \frac{主频}{CPI \times 10^6} $$ |
| MFLOPS | Million Floating Point Operations/Second  | 每秒执行百万次浮点运算   | $$ MFLOPS = \frac{浮点操作数次数}{执行时间 \times 10^6} $$ |
| GFLOPS | Giga Floating Point Operations/Second     | 每秒执行十亿次浮点运算   | $$ GFLOPS = \frac{浮点操作数次数}{执行时间 \times 10^9} $$ |
| TFLOPS | Tera Floating Point Operations/Second     | 每秒执行万亿次浮点运算   | $$ TFLOPS = \frac{浮点操作数次数}{执行时间 \times 10^{12}} $$ |
| PFLOPS | Peta Floating Point Operations/Second     | 每秒执行千万亿次浮点运算 | $$ PFLOPS = \frac{浮点操作数次数}{执行时间 \times 10^{15}} $$ |

### 6.9 其他换算
- 存储容量：$$ 1 KB = 2^{10} B = 1024 B $$，$$ 1 B = 8 bit $$。
- 描述速率、频率时：$$ 1T = 10^3 G = 10^6 M = 10^9 K $$（此处 K、M、G、T 为十进制前缀）。
